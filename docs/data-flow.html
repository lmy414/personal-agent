运行时数据流

总览 架构 数据流 澪号 Harness

一条用户消息的完整路径

从用户按下 Enter 到看见澪号回复，经过 8 个阶段：

阶段 1 — 用户输入

触发者：用户在浏览器输入框键入并按 Enter

数据：{ type: "prompt", text: "读一下 main.cpp 看看渲染循环问题" }

传输：浏览器 → WebSocket → wgnr-pi Server

WebSocket 发送
阶段 2 — 事件触发

触发者：wgnr-pi 将消息转发到 Pi Agent Loop（stdin RPC）

Pi 事件序列：

session_start  → pa-sqlite 连接 DB · pa-usage 打开 DB · pa-mio 重置状态
turn_start     → pa-budget 检查用量（>=80% 黄色 / >=100% 红色）
before_agent_start → pa-mio 组装澪号 Prompt，替换 systemPrompt
阶段 3 — Prompt 组装（pa-mio 核心）

pa-mio 的 assemblePrompt() 函数按 9 个 Slot 顺序拼接：

Slot内容来源类型大小
0元指令（身份 > 工具）meta_instruction.txt静态~80 tokens
1身份层（soul + boundaries）soul.md + boundaries.md静态~1100 tokens
2知识层knowledge.md静态~450 tokens
3[运行环境] Pi 原生提示词 + 工具定义Pi 注入动态变化
4记忆层mio_memories.json 检索按需0~500 tokens
5Chat HistoryPi 管理动态变化
6语言锚硬编码 (~50 tokens)静态~50 tokens
7修正槽计数器触发时注入按需0~80 tokens
8用户消息用户输入动态变化

Slot 0-3 形成 ~2100 tokens 静态前缀，可被 Prompt Cache 命中

阶段 4 — LLM 调用

替换完成：pa-mio 返回 { systemPrompt: 澪号Prompt }，Pi 用其替换系统提示词

API 调用：POST https://api.deepseek.com/v1/chat/completions

消息角色内容
SYSTEM澪号 Prompt（Slot 0-7 拼接）
...Chat History...（多条 USER/ASSISTANT/TOOL）对话上下文
USERSlot 8 — 当前用户消息

模型视角：语言锚（Slot 6）在用户消息之前，是模型生成前最后看到的 SYSTEM 级内容 → 注意力权重最高

阶段 5 — 流式返回 + 计数器检查

流式传输：DeepSeek → Pi → wgnr-pi → WebSocket → 浏览器实时渲染

消息完成：message_end 触发 → pa-mio 跑 5 计数器：

#检测规则触发率
1叠甲≥2 个叠甲词（"我觉得可能""大概是吧"…）中
2emoji 溢出≥3 个 emoji低
3感叹号溢出≥2 个！或 !低
4话痨>200 字中
5亲密溢出<10 轮出现 hhh/嗯嗯/摸摸低

两种情况

全绿 (95%)

直接输出 → 用户看到澪号的回复，无任何延迟感知

触发计数器 (5%)

修正指令注入 Slot 7 → 下轮 before_agent_start 加载 → 澪号带着修正重生成 → 用户无感

阶段 6 — 工具调用（如需要）

如果澪号决定使用工具（如 list_directory / read / write / bash）：

tool_execution_start → pa-mio 显示 "🔧 读取文件中..."
  →
Pi 执行工具（如 read("main.cpp")）
  →
tool_execution_end   → pa-mio 清理状态
  →
Pi 自动触发下一轮 before_agent_start
  →
澪号基于工具返回结果继续回复

关键：当工具返回 >500 字时，pa-mio 在 Chat History 前额外插入语言锚，防止长内容稀释角色风格

阶段 7 — 持久化
事件扩展操作
message_endpa-sqlite写入 messages 表（跳过 toolResult 角色）
message_endpa-usage从 assistant 消息提取 usage → 写入 usage_log 表
阶段 8 — 对话结束 · 记忆提取

触发：agent_end · 条件：对话 >10 条消息 + 本会话尚未提取

流程：

pa-mio 收集完整对话 transcript
  → 独立 DeepSeek API 调用（不受澪号人格限制）
  → System: "从对话中提取值得长期保存的信息，输出 JSON"
  → 得到 [{ "content": "...", "category": "technical", "importance": 3 }, ...]
  → 追加写入 ~/.personal-agent/mio_memories.json（最多 500 条）

下次对话时，Slot 4 检索相关记忆并注入 Prompt，形成"记忆 → 行为 → 新记忆"的闭环
