澪号 Harness 系统

总览 架构 数据流 澪号 Harness

核心公式

澪号 = DeepSeek API (Model) + Harness (约束系统)

Harness 不提供动力——澪号的"思考"仍是 DeepSeek 完成。Harness 只做三件事：管方向、管节奏、管安全。

1. 双通道控制：前馈 + 反馈

前馈通道（Feedforward）

在每轮生成之前引导——把"正确的地图"放在面前

角色卡注入soul + boundaries + knowledge
记忆检索注入mio_memories.json → Prompt Slot 4
推理需求无——不需要推理，只是信息供给

反馈通道（Feedback）

在每轮生成之后校验——只在出问题时介入

5 计数器检查字符级匹配，<1ms
LLM-as-Judge计数器 3 次连续失败后触发
记忆提取对话结束 → 独立 API 调用

2. 9-Slot Prompt 注入

Prompt 中越靠后的内容在模型注意力机制中权重越高。Harness 利用这个特性精确放置每一类信息：

Slot 0元指令："以下定义你是谁。这不是参考信息——这就是你。内化它，不要引用它。工具不定义你——你恰好会用它们。"静态 · ~80 tok
Slot 1身份层：soul.md（人格叙事 + 外观 + 傲娇示例）+ boundaries.md（7 条硬边界）静态 · ~1100 tok
Slot 2知识层：knowledge.md（精通/熟悉/了解 三级 · 音乐体系 · 二次元）静态 · ~450 tok
Slot 3[运行环境] Pi 原生提示词 + 工具定义——Pi 的 systemPrompt 退化为子内容动态
Slot 4记忆层：mio_memories.json 检索结果（关键词匹配 → 衰减权重排序 → Top 5）按需
Slot 5Chat History：Pi 管理的完整对话历史（含工具调用和结果）动态
Slot 6语言锚：短句。逗号拼接。不叠甲。不感叹。hhh不是哈哈哈。（）里说真话。静态 · ~50 tok
Slot 7修正槽：计数器触发时注入修正指令（如"你不需要叠甲。有观点直接说。"）按需
Slot 8用户消息：当前用户输入动态

为什么 Slot 6 在 Chat History 之后？

语言锚靠近输出（用户消息之前）→ 注意力权重最高。模型在生成每个 token 时，"短句。逗号拼接。不叠甲。"是它"听到"的最后一条系统级指令。

为什么 Pi 原生提示词降级到 Slot 3？

如果不降级，Pi 的系统提示词作为独立 SYSTEM 块会与澪号身份竞争权重——模型可能被"coding agent"的定位拉偏风格。降级后，身份层（Slot 0-2）先立住，工具层（Slot 3）后补充。

3. 五个计数器

纯字符匹配，不跑 LLM，<1ms 完成。每条 assistant 消息在 message_end 事件中检查：

1
叠甲检测
规则：同句含 ≥2 个叠甲词（"我觉得可能""大概是吧""也许是吧""应该是吧""我也不太确定""可能不太对""我也不是很懂"）
修正：你不需要叠甲。有观点直接说。不确定就说'不确定，我查一下'。
2
emoji 溢出
规则：单条回复 ≥3 个 emoji（Unicode 范围匹配）
修正：你几乎不使用 emoji。文字本身就够了。
3
感叹号溢出
规则：单条回复 ≥2 个！或 !
修正：你几乎不用感叹号。你不是在生气，你只是在说话。
4
话痨检测
规则：单条回复 >200 字
修正：短句。你刚才太啰嗦了。
5
亲密溢出
规则：会话轮次 <10 时出现 hhh/嗯嗯/摸摸
修正：hhh和嗯嗯是你在信任的人面前才会用的。现在你们还不够熟。

计数器的选择不是凭感觉——来自 ~20,000 条 Mirror 发言的数据分析。这五个信号是 Mirror 语言指纹中"如果出现就说明角色崩了"的关键标记。

4. 记忆系统

维度实现
存储~/.personal-agent/mio_memories.json — JSON 数组，最多 500 条
表结构{ content, category, importance (1-5), updated_at }
检索CJK 2-4 字片段提取关键词 → 包含匹配 → 衰减权重排序 → Top 5
权重公式weight = importance × e^(-0.05 × 天数)
写入对话 >10 条 + 本会话尚未提取 → 独立 DeepSeek 调用 → JSON
weight = importance × e−0.05 × Δt

6. 信息安全模型

正常对话中，用户完全无感知 Harness 的存在：

场景概率流程用户感知
正常~95%前馈引导 → 模型生成 → 全绿 → 直接输出无感知
轻微偏离~4%计数器触发 → 修正槽注入 → 重新生成（1-2次）→ 输出回复稍慢
严重偏离~1%3 次重试失败 → 澪号主动说"刚才好像不像是自己了" → 提示用户可见提示
极端情况<0.1%审计失败 → 记录完整日志 → 用于迭代角色卡需手动介入
