## 数据流

```
Agent 事件 → pa-observe (收集 7 步) → writeTrace()
  ├─ LAST_TRACE: ~/.personal-agent/observe_last_trace.json (实时)
  └─ Session: ~/.personal-agent/observe_traces/<sessionId>.json (持久)

前端 3s 轮询 → GET /api/observe_trace?sessionId=<id>
  ├─ Server 查 observe_traces/<id>.json → 命中返回
  └─ 未命中 → 回退 LAST_TRACE
```

## 已修复

### 1. ✅ traceKey 碰撞 (Root Cause)
- **问题**: `Buffer.from(sessionFile).toString("base64url").slice(0, 32)` —— 所有 session 文件共享相同目录前缀 `C:\Users\Mirror\.pi\agent\sessions\--D--claude-personal-agent--\`，前 32 个 base64url 字符完全相同
- **修复**: `traceKey` 改为直接返回 `sessionId` (ULID UUID)，天然全局唯一

### 2. ✅ buildTrace 崩溃
- **问题**: context 消息的 `m.content` 可能为 `undefined`，`JSON.stringify(undefined)` 返回 `undefined`（非字符串），`.length` 抛异常
- **修复**: `m.content != null ? JSON.stringify(m.content) : "(empty)"`

### 3. ✅ writeTrace 不保存新会话
- **问题**: `if (trace.sessionFile)` —— 新会话 `sessionFile` 可能为 null，导致 session 文件永不写入
- **修复**: 改为 `if (trace.sessionId)` 守卫

### 4. ✅ 启动枚举 LAST_TRACE 竞态
- **问题**: Pi 启动时为每个已加载 session 触发 `session_start`，空会话调用 `writeTrace(empty)` 覆盖 LAST_TRACE，last-writer-wins
- **修复**: `session_start` 空分支只写 session 文件，不碰 LAST_TRACE

### 5. ✅ 前端使用 sessionId 查询
- **问题**: 前端传 `sessionFile`（路径）给 API，server 无法按 session 查找
- **修复**: 前端传 `sessionId`（ULID），server 在 `observe_traces/<id>.json` 中查找

### 6. ✅ 新会话在侧边栏消失 (Windows 路径兼容性)
- **问题**: 创建新会话 → 发消息 → 切换到其他会话 → 新会话从侧边栏消失
- **根因 A**: `server.js` 的 `cwdKey` 计算只替换 `/`，未处理 Windows 的 `\` 和 `:`
  - `D:\claude\personal-agent` → `--D:\claude\personal-agent--`（目录不存在）
  - Pi 实际目录名为 `--D--claude-personal-agent--`
  - `parseSessions()` 找不到目录 → 返回空数组 → 新会话不在 API 响应中
- **根因 B**: `server.js` 使用 `process.env.HOME` 获取主目录，Windows 上可能未设置
  - `join("", ".pi", "agent", "sessions")` → `.pi/agent/sessions`（相对路径，不存在）
- **根因 C**: 前端 `groupByDate()` 中 `+new Date(undefined)` = `NaN`
  - `NaN >= 0` 为 `false` → 会话不落入任何日期分组 → 完全不可见
- **修复**:
  - `cwdKey` 正则扩展：`replace(/[/\\:*?"<>|]/g, "-")` 处理所有 Windows 路径非法字符
  - `process.env.HOME` → `homedir()`（`os.homedir()` 跨平台可靠）
  - `groupByDate()` 添加 `isNaN(raw) ? 0 : raw` 防御
  - `parseSessions()` 的 `lastTimestamp` 添加 `new Date().toISOString()` fallback
