项目定位

Personal Agent 是一个基于 Pi 编码 Agent 框架 + DeepSeek API 的个人 AI 助手桌面应用。

核心特色：通过 澪号 Harness 系统，将通用 AI 模型塑造成具有独立人格、语言指纹和行为边界的角色——澪号。

五层架构速览

层名称实现职责
L5审计层LLM-as-Judge深度检查角色是否 OOC（计数器 3 次失败后触发）
L4记忆层JSON 文件 + 衰减权重跨会话信息检索与注入
L3执行编排层修正槽 + 重试计数器触发 → 修正指令注入 → 重生成
L2工具层Pi 扩展系统文件/数据库/预算/角色控制
L1上下文与约束层角色卡 + 知识库前馈引导——在生成前指明"我是谁"

关键设计决策

  • 1. Pi 原生提示词退化为角色内容——不再是独立 SYSTEM 块，被嵌入 Slot 3 [运行环境] 标签，不与澪号身份竞争权重
  • 2. 双通道控制——前馈通道（角色卡注入，被动引导）+ 反馈通道（5 计数器校验，主动修正）
  • 3. 语言锚位置——放在 Chat History 之后、用户消息之前（Slot 6），获得最高注意力权重
  • 4. 工具不定义身份——元指令明确"你使用 bash 不会变成命令行助手，工具不定义你"
  • 5. 记忆衰减权重——importance × e^(-0.05×天数)，让新旧信息自然竞争
  • 6. Human 是最后防线——95% 对话无感知，4% 触发计数器自动修复，1% 需要用户介入
