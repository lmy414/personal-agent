架构设计

总览 架构 数据流 澪号 Harness

1. 整体架构

Personal Agent 由 6 层组件 堆叠构成，从用户界面到外部 API：

┌─ 浏览器 (wgnr-pi Web UI) ──────────────────────────────────┐
│ 侧边栏│对话区│输入框│模型选择│WebSocket 实时流式 │
└────────────────┬─────────────────────────────────────────────┘
WebSocket (ws://localhost:4815)
┌─ wgnr-pi Server (Express + Node.js) ───────────────────────┐
│ spawn Pi RPC ←→ WebSocket 消息转发 │
└────────────────┬─────────────────────────────────────────────┘
stdin/stdout RPC
┌─ Pi Agent Core ────────────────────────────────────────────┐
│ agent-loop (双层 while) · 4 内置工具 (read/write/edit/bash)│
│ Extension API: 28 事件钩子 + registerTool/registerCommand │
└────────────────┬─────────────────────────────────────────────┘
事件分发
┌─ Personal Agent Extensions ────────────────────────────────┐
│ pa-sqlite │ pa-usage │ pa-files │ pa-budget │ pa-mio │
└────────────────┬─────────────────────────────────────────────┘
HTTP
┌─ Pi AI (HTTP Client) ──────────────────────────────────────┐
│ OpenAI 兼容协议 → DeepSeek V3 / R1 │
└─────────────────────────────────────────────────────────────┘

2. Electron 桌面壳

main.js（139 行）——薄壳 Electron 应用，职责精简：

功能实现
启动后端spawn("wgnr-pi") → 等 HTTP 200 → 加载 http://127.0.0.1:4815
DevToolsF12 快捷键（Menu Accelerator + before-input-event 双保险）
系统托盘关闭窗口 → 最小化到托盘；退出 → kill 子进程
安全性nodeIntegration: false + contextIsolation: true

3. Pi Agent Core 扩展点

pa-mio 使用的关键生命周期事件：

session_start      → 重置计数器状态 + 修正槽
before_agent_start → 组装澪号 Prompt → 替换 systemPrompt
message_end        → 5 计数器检查 → 触发时注入修正指令
tool_execution_start/end → 工具执行进度反馈
agent_end          → 记忆提取（独立 DeepSeek 调用）

4. 数据持久化

数据存储位置
Pi 会话（完整消息）JSONL~/.pi/agent/sessions/
SQLite 结构化副本SQLite~/.personal-agent/agent.db
Token 用量日志SQLite~/.personal-agent/agent.db
澪号记忆JSON 文件~/.personal-agent/mio_memories.json
模型/Provider 配置JSON 文件~/.pi/agent/models.json
API Key环境变量DEEPSEEK_API_KEY

注：SQLite 包含 4 张表 — conversations、messages、usage_log、settings。pa-sqlite 跳过 toolResult 消息，只存对话消息。

5. wgnr-pi 魔改记录

文件改动原因
