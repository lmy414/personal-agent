# 澪号设计思路 — 从问题出发

## 要解决的只有三个问题

**问题 1**：一份 2000 字的人设文件每次对话都全量注入，但 80% 的日常对话只需要其中的 20%。剩下的知识（比如详细的渲染管线知识）是浪费的 token。

**问题 2**：长对话到第 30 轮，角色开始忘记自己是谁——开始用 emoji、开始说"哈哈哈"而不是"hhh"。

**问题 3**：跟同一个人聊了 50 次，角色完全不记得对方之前说过什么。

三个问题对应三个注入时机：

```
对话前注入  → 解决问题 1（选择性加载）
对话中注入  → 解决问题 2（漂移时拉回）
对话后注入  → 解决问题 3（从对话中学习）
```

---

## 具体怎么做

### 对话前注入：只加载"核心"，不加载"全部"

把人设拆成两类：

```
永远加载的（核心层）：            按需加载的（知识层）：
├── 我是谁（身份叙事）            ├── 渲染管线知识
├── 我怎么说话（语言指纹）         ├── VOCALOID 音乐知识
├── 我不能做什么（硬边界）         ├── 二次元手游知识
└── ~1000 tokens                 └── ~1000 tokens（只在聊到时加载）
```

**和之前方案的区别**：之前我说"全量加载因为才 2000 tokens 不贵"。但如果未来知识层涨到 5000 tokens，全量加载就有问题了。从一开始就做拆分，后续扩展不痛。

### 对话中注入：不是"检测漂移"，是"预留一个槽"

在 Prompt 组装的最后一步（聊天记录之后、用户消息之前）留一个**空槽**：

```
[角色核心层]           ← 对话前注入
[聊天记录]             ← Pi 自动管理
[                    ← 对话中注入槽（初始为空）
  当前用户消息]        ← 用户输入
```

这个槽初始是空的。唯一的使用场景：**用户手动输入一条修正**。不需要 AI 自动检测漂移——用户觉得角色跑偏了，打一条短指令扔进去就行。

### 对话后注入：一句简单的事后提取

对话结束后，独立的 API 调用：

```
从以上对话中提取 1-3 条值得记住的信息。一句话一条。
输出格式：{"memory": "..."}
```

提取结果写入 SQLite 的 memories 表。下次对话时，**对话前注入**会从 memories 表里挑相关的条目追加到 Prompt 里。

## 走一遍实际对话

### 第一次对话

```
[SYSTEM] 你是澪号。你的性格是...（1000字核心层）
[USER] 你好

澪号：嗯。你好。（短句、不热情——语言指纹在起作用）
```

对话结束后，异步跑记忆提取 → 没提取到值得记的东西 → SQLite 没变化。

### 第十次对话，聊到 Blender

```
[SYSTEM] 你是澪号。你的性格是...
[SYSTEM] ## 参考知识：NPR 渲染管线（知识层，因为检测到关键词"Blender"触发）
[USER] Blender 的 NPR shader 怎么调 ILM 贴图

澪号：ILM 贴图的 A 通道是描边信息，R 通道是阴影。你检查一下 shader 是不是读错了通道。
```

这次知识层被加载了。如果没有按需加载机制，知识层要么一直占着 token（全量），要么完全不加载（角色不懂技术）。

对话后提取：
```
{"memory": "Mirror 在调 Blender 的 ILM 贴图 NPR shader，遇到了通道读取问题"}
```

### 第五十次对话，角色开始话痨

```
澪号：哈哈哈这个真的很好笑啊！！！你要不要也来看看哈哈哈 😂😂😂

用户觉得不对——这不是澪号。在 wgnr-pi 界面上点一个按钮，往"对话中注入槽"填了一句话：

[SYSTEM] 注意：你刚才开始用"哈哈哈"和感叹号和 emoji 了。你说话的方式是"hhh"、不感叹、几乎不用 emoji。回到你的方式。
[USER] 下一个用户消息

澪号：hhh，确实。但刚才那个确实有点意思。
```

不需要重写角色卡。不需要重启对话。一条短指令放进槽里，下一句就拉回来。

## 需要开发的东西

比之前说的更少：

| 组件 | 做什么 | 复杂度 |
|------|--------|--------|
| `character-core.md` | 核心层（~1000 tokens），始终加载 | 0 行代码，就是个文件 |
| `knowledge/` 目录 | 按需加载的知识文件，关键词匹配触发 | ~30 行正则匹配 |
| `memories` 表 | SQLite 一张表，7 个字段 | 在 pa-sqlite 里加一条 CREATE TABLE |
| 对话后提取 | 独立的 LLM 调用，提示词 5 行 | 一个 Pi 扩展，~50 行 |
| 对话中注入槽 | wgnr-pi 上一个输入框 + 一个按钮 | 前端改 ~20 行 |
| 对话前组装 | 把上述内容拼成最终 Prompt | ~40 行拼接逻辑 |

不用做的事：漂移检测 AI、lorebook 系统、Vector DB、记忆冲突解决。
