好的，我整理了 AI Agent 的六大主流架构模式和对应的适用场景：

1. 单 Agent 架构

一个 LLM 作为中央推理引擎，连接工具和记忆，循环执行直到任务完成。代表产品：Claude Code、Replit Agent。

2. Plan & Execute 架构

将工作分为两阶段——规划器生成步骤列表，执行器逐项执行。无依赖的子任务可并行执行（DAG 结构）。报告显示节省 82% Token。

3. Orchestrator-Worker 架构

编排器 Agent 接收目标，动态分解并委派给专业 Worker，聚合输出。

4. 层级多 Agent 架构

战略层 → 协调层 → 执行层，适合大型企业工作流。

5. "指挥官"架构（2026 企业标准）

  • 指挥官层 — 全局规划，任务流水线化
  • 编排层 — 按任务类型路由到专业 Agent
  • 专家 Agent 层 — 搜索/代码/绘图/合规审计
  • 记忆与资产层 — 向量数据库 + RAG

最小可用的 ReAct Agent 骨架：

import json

class SimpleAgent:
    def __init__(self, model_call, tools=None):
        self.model = model_call
        self.tools = tools or {}
        self.messages = []

    def run(self, user_input, max_steps=10):
        self.messages.append({"role": "user", "content": user_input})
        for step in range(max_steps):
            response = self.model(self.messages)
            if response.get("tool_calls"):
                for tc in response["tool_calls"]:
                    name = tc["function"]["name"]
                    args = json.loads(tc["function"]["arguments"])
                    result = self.tools[name](**args)
                    self.messages.append({
                        "role": "tool",
                        "content": str(result),
                        "tool_call_id": tc["id"]
                    })
            else:
                return response["content"]
        return "Max steps reached."

核心就是 Think → Act → Observe 三步循环。
