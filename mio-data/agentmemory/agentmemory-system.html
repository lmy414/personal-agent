🏛 系统概览

agentmemory 基于 iii engine 三原语（KV / Function / Stream）构建，通过 Hook 自动捕获 AI 编码代理的每个操作，经过压缩、嵌入、索引、固化、遗忘的完整生命周期，实现上下文感知的长期记忆。

核心理念：零外部数据库依赖，所有数据、索引、向量均存储在 iii engine KV 中。

依赖树

iii-sdk          // iii engine 通信（KV / Function / Stream）
zod               // 输出校验
@anthropic-ai/sdk // LLM 压缩（可选）
@xenova/transformers // 本地嵌入（可选，推荐）

iii engine 三原语：
KV       - 持久化键值存储
Function - 远程函数调用
Stream   - 实时事件流

启动流程

1. 加载配置 (loadConfig + loadEmbeddingConfig)
2. 创建 LLM Provider + Embedding Provider
3. registerWorker 连接到 iii engine
4. 初始化 StateKV + VectorIndex + BM25 SearchIndex
5. 注册 50+ Functions (每个 function = 一个 MCP 工具)
6. 从持久化存储恢复索引 (IndexPersistence.load)
7. 启动 REST API + MCP Server + Viewer

🐀 数据模型

Raw RawObservation

原始观测 — Hook 触发时直接存储的原始数据。

interface RawObservation {
  id, sessionId, timestamp, hookType,
  toolName?, toolInput?, toolOutput?,
  userPrompt?, raw, modality?
}
Struct CompressedObservation

LLM 压缩后的结构化观测 — 提取的精华。

interface CompressedObservation {
  id, sessionId, timestamp, type,
  title, subtitle?, facts: string[],
  narrative, concepts: string[],
  files: string[], importance: 1-10,
  confidence?: 0-1
}
Long Memory

长期记忆 — 经过固化后的可持久化知识。

interface Memory {
  id, createdAt, updatedAt,
  type: pattern|preference|architecture|
        bug|workflow|fact,
  title, content, strength: 1-10,
  version, isLatest, supersedes?,
  sourceObservationIds?
}
🎯 Lesson

经验教训 — 带置信度衰减的知识点。

interface Lesson {
  id: fingerprint(content),
  content, context,
  confidence: 0-1, reinforcements,
  decayRate: 0.05,
  lastDecayedAt?, deleted?
}

压缩阶段详解

mem::compress 使用 LLM 将原始观测压缩为 XML 结构化格式：

<compression>
  <type>file_read|command_run|...</type>
  <title>简洁标题</title>
  <subtitle>副标题</subtitle>
  <facts><fact>关键事实</fact></facts>
  <narrative>2-4 句叙述</narrative>
  <concepts><concept>概念标签</concept></concepts>
  <files><file>路径</file></files>
  <importance>1-10</importance>
</compression>

合成压缩（默认，AGENTMEMORY_AUTO_COMPRESS=false）：零 LLM 调用，从原始数据直接提取结构化字段，用于索引。

🧪 四层记忆固化

模仿人类大脑记忆处理方式：

层级类比存储形式
Working (工作记忆)短期记忆RawObservation
Episodic (情景)"发生了什么"CompressedObservation + Summary
Semantic (语义)"我知道什么"Memory + Lesson + GraphNode
Procedural (程序)"如何做"Scripts + Workflow patterns

固化流程 (mem::consolidate)

收集重要观测
importance ≥ 5 → 按 concept 分组
每组 ≥ 3 条 → LLM 合成记忆
取 top 8 → 版本进化
查重 + 版本链 → 级联更新
mem::cascade
版本进化：Jaccard > 0.7 时不覆盖。旧版标记 isLatest: false，创建新版链接 supersedes 链，触发级联更新标记图节点为 stale。

经验教训系统 (mem::lesson-*)

  • 指纹去重 — SHA-256 截断 fingerprintId
  • 强化 — confidence += 0.1 * (1 - confidence)
  • 衰减 — 每周 -decayRate (0.05)，Ebbinghaus 曲线
  • 软删除 — confidence < 0.1 且从未强化
  • 检索 — score = confidence * relevance * recencyBoost

💾 存储层

StateKV

对 iii engine KV 的封装，提供 get / set / delete / list 方法。全部持久化在 iii engine 中。

BM25 SearchIndex

纯内存 BM25 全文搜索引擎：词干提取 | 同义词扩展 | 前缀匹配 (0.5 权重) | CJK 分词 | JSON 序列化

VectorIndex

内存向量索引（余弦相似度）：6 种嵌入提供方 | Base64 序列化 | 维度校验 | Top-K 搜索

IndexPersistence

索引持久化：启动加载 | debounce 保存 (5s) | 删除同步刷新 | 崩溃恢复保护

SearchIndex 内部结构

entries: Map<obsId, { obsId, sessionId, termCount }>    // 文档列表
invertedIndex: Map<term, Set<obsId>>                   // 倒排索引
docTermCounts: Map<obsId, Map<term, freq>>            // 词频表
totalDocLength: number                                   // 平均文档长度
BM25: k1=1.2, b=0.75

去重机制 (DedupMap)

SHA-256(sessionId + toolName + toolInput)
5 分钟时间窗口内自动跳过重复观测

🔎 检索系统

三流混合检索 (HybridSearch)，RRF 融合：

BM25 关键词

词干提取 + 同义词扩展 + 前缀模糊匹配 + CJK 分词

向量语义

余弦相似度，默认 all-MiniLM-L6-v2，支持 OpenAI/Gemini/Voyage/Cohere

知识图谱

BFS 实体匹配 + 关系遍历 + 级联 stale 标记

RRF 融合公式

score = BM25_rank^-1 + VECTOR_rank^-1 + GRAPH_rank^-1   (k=60)
session_diversify: 同 session 最多 3 条结果
BM25_WEIGHT=0.4 | VECTOR_WEIGHT=0.6 | GRAPH_WEIGHT=0.3

mem::smart-search (智能搜索入口)

  • project / cwd 过滤
  • 三种格式：full / compact / narrative
  • Token 预算控制
  • 自动重建空索引
  • 访问日志记录（LRU 淘汰依据）

查询扩展 + 滑动窗口

LLM 将用户查询扩展为多个变体提高召回率；长会话按时间窗口分段压缩索引。

📖 经验教训系统详解

Lessons 系统与主 Memory 互补，专用于高频次、低延迟的"教训"存储和检索。

🔄 生命周期
创建
fingerprint → 强化
confidence +0.1 → 衰减
每周 -0.05 → 软删除
conf < 0.1
📊 核心公式
强化:  conf += 0.1 * (1 - conf)
衰减:  conf -= decayRate * weeks
检索:  score = conf * relevance * 1/(1+days*0.01)
